> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current right-side `MenuDrawer` with a full-viewport mega menu that mirrors the reference screenshot — a navy brand-coloured overlay with the logo centred at top, "CLOSE MENU ×" top-right, and an 8-group, 3-column grid of categorised links (each link prefixed with `> `).

Notes for the implementer:
- **Reused data-attrs**: `data-menu-root`, `data-menu-backdrop`, `data-menu-drawer`, `data-menu-close`, `data-menu-link` — these are queried by `header.client.ts`. Keep the names exactly.
- **`text-surface` on close button + headings**: needed because the overlay is dark navy; if the header's text-colour toggle puts `text-ink` on the trigger, the menu's content is independent of the trigger's colour. Explicit `text-surface` keeps things readable inside the dark overlay regardless.
- **`font-heavy`**: maps to `--font-weight-heavy: 800` from the @theme block.
- **Open/close animation**: drawer enters with `translate-y-2 opacity-0` → `translate-y-0 opacity-100`. The existing `header.client.ts` toggles `translate-x-full` on drawer; we'll update that selector list in Task 2 step 3 below.

- [ ] **Step 3: Update `header.client.ts` to animate the right classes**

The existing `open()` removes `translate-x-full` from the drawer (right-slide pattern); the new drawer uses opacity + translate-y. Edit `src/scripts/header.client.ts` — find the `open` and `close` functions and replace just the drawer class toggles:

Find inside `open()`:
```ts
backdrop.classList.add('opacity-100');
drawer.classList.remove('translate-x-full');
```

Replace with:
```ts
backdrop.classList.add('opacity-100', 'bg-brand-500/40');
backdrop.classList.remove('bg-brand-500/0');
drawer.classList.remove('translate-y-2', 'opacity-0');
drawer.classList.add('translate-y-0', 'opacity-100');
```

Find inside `close()`:
```ts
backdrop.classList.remove('opacity-100');
drawer.classList.add('translate-x-full');
```

Replace with:
```ts
backdrop.classList.remove('opacity-100', 'bg-brand-500/40');
backdrop.classList.add('bg-brand-500/0');
drawer.classList.remove('translate-y-0', 'opacity-100');
drawer.classList.add('translate-y-2', 'opacity-0');
```

The 320ms scroll-lock release timeout in `close()` stays as-is.

Note: with a full-viewport overlay the backdrop is largely covered by the drawer; the small opacity tint just helps the underlying page darken during the cross-fade. Click-on-backdrop still closes the menu because the backdrop captures pointer events on the small areas outside the inner Container.
